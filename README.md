# Crisis Grid

Real-time AI-powered smart city infrastructure monitoring and cascading failure prediction dashboard for Bangalore, India. Monitors power, water, gas, communications, and civil infrastructure with AI-assisted fault analysis powered by Amazon Bedrock.

![Crisis Grid Dashboard](dashboard/public/favicon.svg)

## Features

- Infrastructure Monitoring Dashboard across 5 domains
- AI anomaly detection for sensor Data using ML models
- AI fault analysis via Claude 3 Haiku (Amazon Bedrock)
- Cascade failure simulation for BESCOM power substations
- Disaster mode overlays (flood, earthquake, fire, etc.)
- Incident tracking with severity classification
- Historical analytics and failure prediction
- SNS citizen alert notifications for critical faults

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React 19, Vite, Recharts, Leaflet |
| Backend | AWS Lambda (Python 3.12) |
| AI | Amazon Bedrock — Claude 3 Haiku |
| Database | Amazon DynamoDB |
| Alerts | Amazon SNS |
| API | Amazon API Gateway (HTTP API) |
| ML | scikit-learn, XGBoost, Random Forest |

## Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
cd dashboard && npm install
```

### 2. Deploy AWS infrastructure
```bash
python deploy/setup_aws.py
```
Provisions Lambda, API Gateway, DynamoDB, SNS. Writes `dashboard/.env` automatically.

### 3. Train ML models (optional)
```bash
python pipeline/preprocess.py
python models/train_models.py
```

### 4. Run locally
```bash
cd dashboard && npm run dev
```
Open http://localhost:5173

### 5. Build for production
```bash
cd dashboard && npm run build
```

## Environment Variables

Create `dashboard/.env` (auto-generated by `setup_aws.py`):

```env
VITE_API_URL=https://<api-id>.execute-api.us-east-1.amazonaws.com
VITE_AWS_REGION=us-east-1
VITE_API_KEY=<your-secret-key>
```

> Never commit `.env` or `deploy/aws_config.json` — both are gitignored.

## Security

- API key authentication on every request (`x-api-key` header)
- CORS locked to configured frontend origin
- Input validation and allowlisting on all Lambda inputs
- Secrets stored in Lambda environment variables, never in source code

## AWS IAM Requirements

Your IAM user needs:
- `AWSLambda_FullAccess`
- `AmazonAPIGatewayAdministrator`
- `AmazonDynamoDBFullAccess`
- `AmazonBedrockFullAccess`
- `AmazonSNSFullAccess`
- `IAMFullAccess`
- `AmazonS3FullAccess`

Enable Bedrock model access for `anthropic.claude-3-haiku-20240307-v1:0` in the AWS Console before deploying.
